Desenvolvido por Daniela Bonfim

Requisitos

Este arquivo visa documentar a proposta do desenvolvimento de um sistema de armazenamento de dados para escritórios de Arquitetura.

Restrições

A interface do projeto deve ser totalmente voltada à WEB.

Premissas

O cliente deverá disponibilizar tanto um servidor web, quanto uma plataforma de armazenamento em nuvem.

Requisitos Funcionais

Sigla Descrição Tipo
RQ01 O sistema deve permitir o cadastro das principais informações dos clientes Essencial
RQ02 O sistema deve permitir o cadastro das principais informações dos projetos Essencial
RQ03 O sistema deve permitir o cadastro das principais informações dos prestadores de serviços (parceiros) Essencial
RQ04 O sistema deve permitir o cadastro das principais informações dos funcionários Essencial
RQ05 O sistema deve ser acessado através de login e senha Essencial
RQ06 O sistema deve relacionar os seus dados:os clientes com seus repectivos projetos; os fornecedores com os projetos para os quais foram contratadados; os funcionarios com os projetos nos quais trabalharam. Essencial
RQ07 O sistema deve permitir o envio de arquivos de projetos com as principais extensões do ramo: CAD(.dwg), Sketchup (.skp), Revit (.rvt), ArchiCad (PLN), Excel (.xls), além de imagens (.jpeg e .png) e PDF. Importante
RQ08 O sistema deve permitir o cadastro de "abas" na sessão de arquivos de projetos de acordo com a fase especificada. A interface de acesso aos arquivos, também, a data de envio, o seu título e o nome de quem realizou o upload. Essencial
RQ09 O sistema deve vincular aos projetos os orçamentos recebidos dos fornecedores cadastrados Importante
RQ010 O sistema deve permitir a exclusão e edicação de dados cadastrados Essencial
RQ011 O sistema deve alertar, através de e-mail, os funcionários quanto à datas de aniversário dos clientes Desejável

Requisitos Não Funcionais

Sigla Descrição
RNF01 O sistema deve operar em interface restrita à WEB.
RNF02 A sincronização dos dados deverá ser realizada em tempo real.
RNF03 O sistema deve possuir segurança dos dados cadastrados.
RNF04 O sistema deve ser robusto para atender à demanda simultânea e escalável para futuras implementações.

Fluxo

O sistema foi construído com a característica de que um cliente pode não possuir projetos, porém, todos os projetos devem pertencer a um, e somente um, cliente. Portanto, é necessário que se siga o fluxo de primeiro inserir o cliente para, então, adicionar o projeto.
